Задание: Напишите веб-приложение на ASP.NET MVC для редактирования одной таблицы в БД. Структура таблицы на Ваш вкус‚ но обязательно чтобы в ней было хоть одно поле-справочник‚ ссылающееся на другую нередактируемую таблицу. Особенность - в списке записей данные должны загружаться динамически (для ASP.NET MVC - AJAX ом при помощи jQuery)

Мое решение состоит из пяти проектов.

OnlineShop.Admin
OnlineShop.DataAccess
OnlineShop.Domain
OnlineShop.Services
OnlineShop.WebInterface

Особых комментариев к проектам, думаю, не требуется.

В соответствии с заданием проект OnlineShop.WebInterface выводит на экран таблицу брендов, при клике на бренд подгружается ассортимент продукции, при клике на продукт выводится инпут для редактирования наименования.
Кнопка «Сохранить» сохраняет изменения в объект на клиенте и в базу данных на сервере.

Проект OnlineShop.Admin сейчас позволяет только загрузить первоначальные данные из файла.
Данные в файле – это реальный прайс от компании BERG.

Для запуска проекта нужно запустить проект, пересобрать проет и восстановить пакеты. 
Далее необходимо раскомментировать одну из строк подключения в файле Web.config.
•	Если раскомментировать строку «Для работы с базой данных на сервере», то будет создана новая БД на вашем локальном компьютере. В этом случае ее необходимо заполнить данными.
Для этого нужно запустить проект OnlineShop.Admin и нажать «Загрузить данные», данные грузятся очень долго т.к. мною сознательно выбран самый не эффективный способ загрузки.
В реальном приложении, я предпочитаю загружать данные хранимыми процедурами, создавая при этом на сервере табличные типы.
Управление базой данных реализую через WebService.

•	Если раскомментировать строку «Для работы с локальной базой данных», то проект должен запуститься и все указанные в задании функции должны работать.
